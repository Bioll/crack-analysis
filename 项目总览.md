# 项目总览 · 这个文件夹做了什么

裂纹的 **生成 → 分析 → 预测** 全流程，外加对一批经典论文的复刻和高难案例尝试。
所有仿真都用相场法 (phase-field) 等自有代码、在本地 WSL2 + FEniCSx 环境跑出。

> ⚠️ 数据均为**仿真/解析合成数据**（外加少量公开真实数据集），结果定位为**方法流程演示**，
> 非经实验验证的工程预测器。神经网络在干净合成数据上精度虚高，不代表真实疲劳预测能力。

---

## 一、主线流程（三阶段）

| 阶段 | 做的事 | 脚本 | 关键结果 |
|------|--------|------|---------|
| **1 生成** | 相场法 SENT 拉伸生成裂纹 | `src/phase_field_sent.py` | 峰值力 629N、水平裂、载荷-位移曲线 |
| **2 分析** | 提取几何/力学特征+打标签 | `src/analyze_crack.py` | 长度/CMOD/曲折度/分叉/方向/K_c + 萌生/等级/失稳 |
| **3 预测** | 神经网络预测疲劳剩余寿命 RUL | `src/train_nn.py` | R²=0.9998（合成数据, 虚高）|

阶段2 做过三处精度修正：① 预制裂纹改 H 场播种（长度 0.92→0.54≈真值0.5）；
② "宽度"改测物理 CMOD；③ K 用方法B `K_c=√(E'·Gc)` 标定。

## 二、疲劳（两条路）

| 方法 | 脚本 | 结果 |
|------|------|------|
| Paris 公式（快/经验） | `src/paris_fatigue.py` | a-N 曲线 + da/dN 直线(斜率m=3) + 寿命 |
| 相场疲劳（高保真/Carrara 2020） | `src/phase_field_fatigue.py` | 潜伏期+加速扩展, 寿命~1711循环 |

阶段3 数据集由 `src/fatigue_dataset_gen.py` 扫多工况生成。

## 六、环境（项目本地, 不进全局）

WSL2 Ubuntu + Miniconda，两个**项目专用前缀环境**（`~/cracks_env/`）：
- `~/cracks_env/fenicsx` — FEniCSx/dolfinx 0.11 + scikit-image（仿真/分析）
- `~/cracks_env/ml` — PyTorch CPU + pandas + scikit-learn（神经网络）

运行示例：
```bash
wsl bash -lc "source ~/miniconda3/etc/profile.d/conda.sh; conda activate ~/cracks_env/fenicsx; python src/phase_field_sent.py"
```
conda 频道用清华镜像（国内网络）。详见 `environment_*.yml`。
